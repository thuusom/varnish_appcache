Varnish authentication example

This is an example of how varnish can be configured to call an authentication url on requests that are to be secure and still cachable.

Authentication calls

The authentication request is just /authentication.html which always returns 200. The protected but cachable resource is secure and in the access log you should see a call to authentication.html every time there is a call to secure.html, but the response isn't cached.

Authenticated calls - cached

The calls to Secure Cached is doing the same as above, but the response is cached which you should see on the x-cache header in the response.

Unauthenticated calls cached and not cached

This is standard varnish, but calls to Unsecure are not cached and Unsecure cached are cached. Both are not requirering authentication calls.

Note!

When accessing this directly in nginx all requests are uncached and unsecure!
